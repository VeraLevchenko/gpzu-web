# План реализации кнопки MapInfo (РРР)

Проект: `/root/projects/gpzu-web`

Документ предназначен для передачи разработчику/ИИ (Claude Code ) для реализации механизма формирования и записи контура в MapInfo слой.

---

## 1. Цель

Реализовать кнопку **MapInfo** в модуле РРР, которая:

- формирует геометрию участка из XML;
- создаёт временный слой через MIF/MID → TAB (по механизму Gradplan);
- выполняет merge во внешний слой;
- удаляет временные файлы;
- корректно заполняет атрибуты.

---

## 2. Основной слой

Путь:

```
/root/projects/gpzu-web/backend/gis_layers/РэРэРэ.TAB
```

Слой тяжёлый, работа должна выполняться без полной загрузки в память.

---

## 3. Источники данных

### 3.1 Геометрия

- Источник: XML ;
- Поддержка:
  - multipart;
  - отверстия (holes);
- Использовать логику Gradplan (проекции, перестановка координат).

### 3.2 Атрибуты

Источник: карточка РРР.

Проверять перед созданием слоя:

| Поле |
|------|
| Заявитель |
| Площадь |
| Вид_Объекта |
| Наименование |
| Входящий_номер |
| Входящая_дата |

Если поле пустое — операция запрещена.

---

## 6. Правила заполнения атрибутов

- Все поля должны быть записаны;
- Обязательные — из карточки;
- Необязательные — пустые:
  - Char → ""
  - Date → NULL.



## 8. Алгоритм работы

### Шаг 1. (Реализовано)

- Загрузка карточки;
- Загрузка геометрии 

### Шаг 2. Валидация

- Проверка обязательных полей;
- Проверка геометрии.

### Шаг 3. Создание временного слоя

```
/tmp/mapinfo/<card_id>/<timestamp>/
```

- Формирование MIF/MID;
- Конвертация в TAB (по Gradplan).

### Шаг 4. Merge

Основной вариант: append.

Fallback: пересборка слоя.

С блокировкой файлов.

### Шаг 5. Очистка

Удаление временных файлов.

### Шаг 6. Ответ API

Возврат статуса операции.

---

## 9. Merge-стратегия

### Вариант A — Append

- Открытие TAB в update-режиме;
- CreateFeature.

### Вариант B — Rebuild

1. Создать main_new.tab;
2. Скопировать main → main_new;
3. Добавить tmp → main_new;
4. Backup;
5. Rename.
