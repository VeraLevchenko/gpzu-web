# Создать файл документации правил генерации решений РРР

## Контекст
Пользователь попросил «написать все правила подготовки документа, все условия и сохранить в файл».
Нужно создать `backend/docs/rrr_decision_rules.md` — подробную справку по всем условиям и правилам,
которые определяют содержание решения о разрешении на размещение.

## Целевой файл
`backend/docs/rrr_decision_rules.md` (создать новый)

## Структура документа

### 1. Виды объектов и флаги (из object_types.json)
Таблица 38 видов с колонками: №, Краткое наименование, is_linear, is_tech_connection, is_nto, has_payment_default, max_term_years

Специальные правила платности по наименованию:
- п.4: платно если имя содержит «парковка»/«стоянка»/«автостоянка»
- п.6: платно только если «нефтепровод»
- п.19: платно только если «кафе»

### 2. Правовые основания (раздел «На основании»)
**Всегда:**
- ст.39.36 ЗК РФ
- ПП РФ от 03.12.2014 №1300
- ПП КО от 01.07.2015 №213
- Правила благоустройства №16/198
- Положение о Комитете

**Если is_linear (пп. 1,2,3,5,6,7,8,32):** + ст.51 ГрК РФ + ПП РФ от 12.11.2020 №1816
**Если is_nto (пп. 19,23,24,25,36):** + ПП администрации г. Новокузнецка от 11.04.2018 №63
**Если proezd_agreement (п.9, вводится вручную):** + согласование на примыкание

### 3. Платность и расчёт платы
Платные по умолчанию — все объекты, кроме пп. 4, 4.1, 6, 12, 13, 18, 19, 21, 26, 27, 29.
Специальные случаи:
- п.4: платно если «парковка»/«стоянка»/«автостоянка» в названии
- п.6: платно если «нефтепровод» в названии
- п.19: платно если «кафе» в названии

**Формула стандартная (все платные кроме п.5):**
П_год = Су × Нст × S = 899,98 × 0,015 × S
П_итого = П_год × (дней_периода / 365)

**Формула ЛЭП (только п.5):**
П_год = 10,06 × S × 1,04
П_итого = П_год × (дней_периода / 365)

Период = дата решения → дата окончания.
Если has_payment → добавляется п.3 с реквизитами + Приложение №2 «РАСЧЁТ ПЛАТЫ»

Реквизиты: ИНН 4217080382, КПП 421701001, БИК 013207212
Получатель: УФК по Кемеровской области – Кузбассу (Комитет градостроительства...)

### 4. Нумерация пунктов (динамическая)
Счётчик _n стартует с 3.
- Если has_payment → п.3 = «Оплата», _n становится 4
- Далее последовательно: Прекращение → Ликвидация → Земляные работы → Геодезия → РСО → Права 3-х лиц → Вырубка → Очистка
- Если is_tech_connection → вставляется пункт «Техприсоединение»
- Затем: Окончание срока → Содержание → Восстановление покрытия → Контроль

### 5. Прекращение действия решения
Базовые пп.1-4 всегда: срок, ликвидация, предоставление ЗУ, ненадлежащее использование.
Если has_payment → доп. пункт «невнесение платы»
Если is_nto → доп. пункт «отсутствие свидетельства аттракциона»

### 6. Подпункты п.8 (РСО)
Всегда: пп.1-11
Если is_tech_connection: п.12 = «Техприсоединение»
Если has_red_lines (красные линии > 0 кв.м): следующий пункт = «Проект планировки дорожной сети»

### 7. Срок действия
max 5 лет: пп. 1, 2, 3, 5, 6, 7, 11
max 3 года: все остальные
Формат: «36 (тридцать шесть) месяцев»

### 8. Заявитель
ЮЛ: наименование + ИНН + ОГРН + адрес → «(далее — Заинтересованное лицо)»
ФЛ: ФИО + паспорт + адрес → «(далее — Заинтересованное лицо)»
