# Фикс: дублирующиеся секции РРР в пространственном анализе

## Контекст

После добавления секции `prev_decisions` («Ранее выданные решения РРР») в UI появились две секции с похожими названиями:

- `key: 'prev_decisions'` → «Ранее выданные решения РРР (Разрешение на использование ЗУ)» — новая, анализирует слой `Разрешение на использование ЗУ.TAB` (поля: вид объекта, номер решения, дата, срок, заявитель)
- `key: 'rrr'` → «РРР (ранее выданные решения)» — старая, анализирует слой `RRR_APPLICATIONS` / `РэРэРэ.TAB` (поля: вх. номер, вх. дата, заявитель, наименование)

Старая секция `rrr` содержит **входящие заявления**, а не выданные решения — но её метка вводит в заблуждение. Дублирования данных нет, только путаница в названиях.

## Изменение — `frontend/src/components/RRR/RRRCard.jsx`

**Файл:** `frontend/src/components/RRR/RRRCard.jsx`, строка ~386

Переименовать метку старой секции `rrr`:

**Было:**
```jsx
label: 'РРР (ранее выданные решения)',
```

**Станет:**
```jsx
label: 'Заявления РРР (входящие)',
```

## Проверка

Открыть вкладку «Пространственный анализ» — секции должны называться:
- «Заявления РРР (входящие)» — входящий номер, дата, заявитель, наименование
- «Ранее выданные решения РРР (Разрешение на использование ЗУ)» — вид объекта, номер решения, дата решения, срок, заявитель
