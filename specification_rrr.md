# ТЕХНИЧЕСКАЯ СПЕЦИФИКАЦИЯ
## Автоматизация услуги "Выдача решения о разрешении размещения объектов"

**Дата:** 31 января 2026  
**Версия:** 1.0  
**Статус:** Финальная спецификация для разработки

---

## 1. ОБЩЕЕ ОПИСАНИЕ УСЛУГИ

### 1.1 Правовая основа
- **Земельный кодекс РФ** ст. 39.36
- **Постановление Правительства РФ от 03.12.2014 № 1300** - перечень из 38 видов объектов
- **Постановление Коллегии Администрации Кемеровской области от 01.07.2015 № 213** - Положение о порядке и условиях размещения

### 1.2 Суть услуги
Выдача разрешения на размещение объектов на землях государственной или муниципальной собственности **без предоставления земельных участков** и установления сервитутов.

### 1.3 Участники процесса
- **Заявитель** - физическое или юридическое лицо
- **Специалист отдела** - единственный пользователь системы, обрабатывает заявления

### 1.4 Сроки оказания услуги
- **Общий срок:** 30 рабочих дней
- **Электричество (п.5):** 5 рабочих дней
- **Газопроводы (п.6):** 7 рабочих дней
- **Нефтепроводы (п.6):** 15 рабочих дней

### 1.5 Результат
- ✅ **Решение о разрешении размещения** (DOCX)
- ❌ **Отказ в разрешении размещения** (DOCX) - _раздел в разработке_

## 2. БИЗНЕС-ПРОЦЕСС

### 2.1 Схема процесса

```
┌─────────────────────────────────────────────────┐
│  ЭТАП 1: РЕГИСТРАЦИЯ ЗАЯВЛЕНИЯ                  │
├─────────────────────────────────────────────────┤
│ • Выбор способа подачи (бумага/ЕПГУ)            │
│ • Загрузка файлов (DOCX + XML или только XML)  │
│ • Парсинг данных                                │
│ • Заполнение обязательных полей вручную         │
│ • Автоматический пространственный анализ        │
│ • Сохранение в БД                               │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│  ЭТАП 2: РАБОТА С ЗАЯВЛЕНИЕМ (гибкий порядок)   │
├─────────────────────────────────────────────────┤
│ • Редактирование полей (в любой момент)         │
│ • Пересчет пространственного анализа            │
│ • Создание карточки в Kaiten                    │
│ • Добавление объекта в слой MapInfo             │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│  ЭТАП 3: ПРИНЯТИЕ РЕШЕНИЯ                       │
├─────────────────────────────────────────────────┤
│ • Формирование решения (DOCX) → скачивание      │
│ • ИЛИ Формирование отказа (в разработке)        │
└─────────────────────────────────────────────────┘
```

### 2.2 Способы подачи заявления

#### Вариант А: Бумажное заявление + XML
1. Заявитель приносит бумажное заявление + диск с XML схемой границ
2. Специалист загружает только XML → парсинг координат
3. Все остальные данные вводятся вручную

#### Вариант Б: Через ЕПГУ
1. Заявитель подает через ЕПГУ (DOCX + XML)
2. Специалист загружает оба файла → автоматический парсинг
3. Вручную заполняются только 2 поля (Вид объекта, Наименование объекта)

## 3. ИСТОЧНИКИ ДАННЫХ

### 3.1 Классификация полей по источникам

| Источник | Описание | Поля |
|----------|----------|------|
| **Заявление (DOCX)** | Парсинг или ручной ввод | ФИО/Название орг, ИНН, ОГРН, Адрес, Паспорт, Вх.№, Вх.дата, Срок действия |

| **Расчетные** | Автоматически | Срок оказания услуги, Дата окончания |
| **Пространственный анализ** | Автоматически (15 слоев) | Кварталы, ОКС, ЗОУИТ, Красные линии и др. |
| **Пустые** | Опционально | ЗОУИТ (доп.), Примечание |

**Всего 38 пунктов + подпункты:**

| № | Краткое название | Срок |
|---|------------------|------|
| 1 | Подземные линейные сооружения | 30 дн |
| 2 | Водопроводы и водоводы | 30 дн |
| 3 | Линейные сооружения канализации | 30 дн |
| 4 | Элементы благоустройства территории | 30 дн |
| 4.1 | Пандусы для маломобильных групп | 30 дн |
| **5** | **Линии электропередачи до 35 кВ** | **5 дн** |
| **6** | **Трубопроводы (газо-, нефте-, до 1,2 Мпа)** | **7/15 дн** |
| 7 | Тепловые сети | 30 дн |

**Формат в интерфейсе:**

### 6.3 Особенности расчета

#### Красные линии (ПП УДС)
Вычисляется **2 значения:**
- Площадь участка **внутри** красных линий (intersection)
- Площадь участка **за границами** красных линий (difference)

#### Множественные пересечения
Если участок пересекается с **несколькими** объектами одного типа → все добавляются в JSON массив

### 6.4 Момент выполнения
- **Автоматически** при регистрации заявления (после загрузки XML)
- **Вручную** по кнопке "Пересчитать" (полная перезапись данных)
- **Сохранение** результатов в БД (не пересчитывать при каждом открытии)

## 7. ИНТЕГРАЦИЯ С KAITEN

### 7.1 Параметры карточки

| Параметр | Значение |
|----------|----------|
| **API endpoint** | Kaiten API (ключ есть) |
| **Пространство/Доска** | ID есть (уточнить в конфигурации) |
| **Название карточки** | `{вх_номер} {заявитель}` <br>Пример: `123 ООО "Газпром"` |
| **Ответственный** | Текущий пользователь (кто создал заявление) |
| **Колонка (статус)** | Определить позже |

### 7.2 Поля карточки Kaiten

**Обязательные поля для создания:**
1. Наименование юридического/физического лица
2. Вх. Номер
3. Вх. Дата (формат `01.01.2025`)
4. Наименование объекта
5. Срок оказания услуги (дата)

### 7.3 Логика создания
- Кнопка **"Создать карточку в Kaiten"** активна при заполнении всех обязательных полей
- Можно создавать **повторно** (обновление данных)
- После создания карточки ID сохраняется в БД

## 8. ИНТЕГРАЦИЯ С MAPINFO

### 8.1 Слой для добавления объектов

**Файл:** `T:\NOVOKUZ\Разрешение на использование ЗУ.TAB`

**Структура полей (из Лист2 Excel):**
```
ID (автоинкремент)
Заявитель (текст)
Номер_решения (текст)
Дата_Решения (дата)
Площадь (float)
Дата_окончания (дата)
Вид_объекта (текст)
Наименование (текст)
Местоположение (текст)
Примечание (текст)
Кадастровый_номер (текст)
ДатаЗанесения (дата)
+ геометрия (Polygon)
```

### 8.2 Поля для MapInfo (всего 23 поля)

**Обязательные для добавления:**
1. Наименование юр/физ лица
2. Паспорт
3. Вх. Номер, Вх. Дата
4. **Вид объекта** (вручную)
5. **Наименование объекта** (вручную)
6. Срок действия решения
7. Площадь, Местоположение, Координаты (XML)
8. Дата окончания (расчет)

**Необязательные:**
- Номер решения, Дата решения (заполняются позже)
- ЗОУИТ, Примечание (могут быть пустыми)

### 8.3 Процесс добавления
1. Парсинг координат из XML
2. Создание MIF/MID файлов (используя существующий модуль `mif_writer.py`)
3. Конвертация MIF → TAB (через MapInfo или ogr2ogr)
4. Добавление объекта в слой `Разрешение на использование ЗУ.TAB`

### 8.4 Логика
- Кнопка **"Добавить в MapInfo"** активна при заполнении обязательных полей
- Можно добавлять **повторно** (обновление геометрии и атрибутов)
- После добавления в слой флаг сохраняется в БД

### 9.5 Логика кнопок

#### "Обновить из источников"
- Показать диалог загрузки новых файлов
- Или заново распарсить существующие
- Обновить все поля из файлов

#### "Редактировать"
- Все поля доступны для ручного изменения
- Сохранение по кнопке "Сохранить"

#### "Пересчитать"
- Полная перезапись пространственного анализа
- Обновление расчетных полей (срок, даты)

#### "Создать в Kaiten"
**Активна если заполнены:**
- Наименование юр/физ лица ✓
- Вх. Номер ✓
- Вх. Дата ✓
- Наименование объекта ❌
- Срок оказания услуги ✓

**Действие:** Создание/обновление карточки в Kaiten

#### "Добавить в MapInfo"
**Активна если заполнены:**
- Все базовые данные ✓
- Вид объекта ❌
- Наименование объекта ❌
- Координаты ✓

**Действие:** Добавление объекта в TAB слой

#### "Сформировать решение"
- Генерация DOCX по шаблону
- Скачивание файла
- БД/Kaiten/MapInfo НЕ обновляются

#### "Сформировать отказ"
- **Статус:** В разработке
- Показать заглушку "Раздел в разработке"

## 11. ФОРМИРОВАНИЕ РЕШЕНИЯ (DOCX)

### 11.1 Шаблон документа

**Файл:** `backend/templates/rrr/decision_template.docx`

**Плейсхолдеры (25+ полей):**
- `{{OUT_NUMBER}}` - исходящий номер решения
- `{{OUT_DATE}}` - дата решения
- `{{APP_NUMBER}}` - входящий номер заявления
- `{{APP_DATE}}` - дата заявления
- `{{APPLICANT}}` - заявитель (юрлицо или физлицо)
- `{{INN}}`, `{{OGRN}}` - для юрлиц
- `{{PASSPORT}}`, `{{ADDRESS}}` - для физлиц
- `{{OBJECT_TYPE}}` - вид объекта
- `{{OBJECT_NAME}}` - наименование объекта
- `{{AREA}}` - площадь
- `{{LOCATION}}` - местоположение
- `{{TERM_MONTHS}}` - срок действия (месяцы)
- `{{END_DATE}}` - дата окончания действия
- `{{QUARTERS}}` - кварталы
- `{{CAPITAL_OBJECTS}}` - ОКС (список)
- `{{ZOUIT}}` - ЗОУИТ (список)
- `{{RED_LINES_INSIDE}}`, `{{RED_LINES_OUTSIDE}}` - площади

### 11.2 Генерация
- Библиотека: `python-docx`
- Модуль: `backend/generator/rrr_decision_builder.py`
- Логика: замена плейсхолдеров + вставка таблиц для множественных данных
- Сохранение: временный файл → скачивание
